Scan for files and applications:

Scan-ForAppOrFilepath will look for files/folders/applications with names that contain the value input for the $Item parameter.

The $ShowMisses parameter lets the user specify whether they want computers that did not have any matching files/folders/applications to appear in the result list.

  • 'n' will not show misses - computers that don't have matching items won't show in spreadsheet.
  • 'y' will cause misses to appear in spreadsheet.

Example of usage: using 'Respondus' as the $Item in an application search will return a row for each match on a computer.

  • Computers with multiple rows have multiple respondus installations (msi, exe)
  • Sort the spreadsheet by UninstallString to see Respondus .exe installations that can be removed.

Sophos Endpoint Self Help scan:

You can scan the network to make sure that all responsive devices have the Sophos Endpoint Self Help application installed.

Presence of the application isn't a foolproof indication of a compliant computer, but it's strong supporting evidence.

You can sort the resulting spreadsheet by version - most of the time if there are 1 or several machines with lesser version of Self Help - they may still be compliant in terms of Sophos Admin Console, but they will fail self-help checks in my experience. Most of the time, these were failures of Sophos AutoUpdate.

Sorting the spreadsheet by Application name will show you computers that have no Sophos or older versions - these can be resolved by running the script from our MDT server to install Sophos (or the menu's PSADT script).

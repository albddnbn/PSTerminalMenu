'Canned' Response Functionality

Author : abuddenb
Updated : January 20, 2024

'Canned responses' can be useful for a variety of purposes - most notably - you can create templates for common responses to issues that come up frequently to help you respond more efficiently. For ex: if there are a lot of users dealing with a calendar-related issue in Outlook that's able to be resolved through same process - you can create a canned response here to include a variable for the user's name.

Ideas for improvement - ability to insert pictures into canned responses easily when using New-CannedResponse.

The terminal menu can copy custom 'canned responses' to the user's clipboard, while prompting the user to enter values for any variables contained in the response file.

The menu is also able to generate new canned response files based on user input.

Basic canned response example:

A canned response is an HTML file that you can quickly personalize and copy to clipboard, to efficiently respond to common IT Support Tickets.

Once you've copied the response to clipboard, you have to click the 'code' button in OSTicket before pasting if the canned response contains HTML.

Text content can be pasted directly into response textbox without clicking the <> (code) button beforehand.

Variables:

The most common variable you'll see in the canned responses that have already been created is the Ticket Owner variable, often called 'owner'.

It appears as (($owner$)) in the response files (located in the $env:PSMENU_DIR/canned_responses directory).

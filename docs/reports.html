Report Creation:

Author : abuddenb
Updated : January 20, 2024
CSV and XLSX reports are generated at the end of many of the functions pre-loaded in the menu.
The ImportExcel module is used to create .xlsx files, the menu attempts to install/import the module. If it errors, only a .csv file will be generated.

File name/path creation:

Any function that creates .csv/.xlsx reports will use the Get-OutputFileString function in the 'utils' folder to create an output filepath (without file extension).

All functions that generate .csv/.xlsx reports should have an $OutputFile variable.

  • If '' (blank) is submitted as a value for $Outputfile - the function will create an output filepath using default filenames.

    • Ex: Get-CurrentUser 's default filename follows the format: 'CurrentUsers-<thedate>'

  • If 'n' is submitted - no reports are created, the function will only print results to terminal.
  • Any other value submitted for $Outputfile, will be used in the creation of the output filepath.

If the output filepath already exists, functions will append a number to the end of the filepath until there is no existing file/folder with the path.

The utility functions' code can probably be condensed significantly or otherwise improved.
